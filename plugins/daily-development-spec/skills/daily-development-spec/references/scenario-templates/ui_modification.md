# Scenario Template: ui_modification

## Goal
Deliver UI changes that preserve usability, consistency, and accessibility.

## Requirement Checklist
- Target screens/components
- Visual/interaction changes
- Responsive behavior expectations
- Accessibility and keyboard support requirements
- Cross-browser/device validation scope

## Deliverables
- Updated UI components/styles
- Screenshots or before/after notes
- Component/unit tests where applicable
- Integration or E2E checks for user flow
- Progress update in `docs/progress/MASTER.md`

## Acceptance Criteria
- UI behavior matches requested interaction
- No layout regressions on supported viewports
- Accessibility baseline remains acceptable
- Tests pass for impacted areas
