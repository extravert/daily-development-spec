# Scenario Template: new_feature

## Goal
Implement incremental business functionality with clear scope, tests, and rollout readiness.

## Requirement Checklist
- User/business value statement
- In-scope modules and exclusions
- Domain rules and edge cases
- Data model impact
- Rollback or fallback strategy

## Deliverables
- Feature implementation
- Unit tests for business rules
- Integration tests for module interactions
- Updated docs for behavior and constraints
- Progress update in `docs/progress/MASTER.md`

## Acceptance Criteria
- Feature behavior matches agreed success criteria
- Edge cases are covered by tests
- No regressions in related flows
- Delivery notes include known limitations
