# Document Templates Reference

This file contains standard templates for documents generated by the Daily Development Specification workflow.

---

## Directory Structure

```text
docs/
└── progress/
    ├── MASTER.md
    ├── phase-0-<short-name>.md
    ├── phase-1-<short-name>.md
    ├── phase-2-<short-name>.md
    ├── phase-3-<short-name>.md
    └── completed/
```

---

## Phase Design Note Template (`phase-1-<short-name>.md`)

```markdown
# Phase 1: Compliance Check & Architecture Design

## Requirement
- **Task**: One-line requirement statement
- **Scenario**: new_interface | new_script | new_feature | ui_modification | performance_optimization
- **Scope**: In-scope modules/files
- **Out of Scope**: Explicit exclusions

## Compliance Checklist
- [ ] Naming/style conventions reviewed
- [ ] Parameter handling approach confirmed
- [ ] Error handling approach confirmed
- [ ] Security baseline reviewed
- [ ] Testing approach confirmed

## Architecture Design
- **Affected Files**:
  - `path/to/file`
- **Contracts**:
  - Input:
  - Output:
  - Error cases:
- **Implementation Order**:
  1. ...
  2. ...

## Risks and Mitigation
- Risk:
- Mitigation:

## Confirmation
- [ ] User reviewed and approved Phase 1 design
```

## MASTER.md (Master Control File)

```markdown
# [Task Name] -- Progress Tracker

> **Task**: One-line description
> **Started**: YYYY-MM-DD
> **Last Updated**: YYYY-MM-DD

## References
- [Phase 0 Details](./phase-0-<name>.md)
- [Phase 1 Details](./phase-1-<name>.md)
- [Phase 2 Details](./phase-2-<name>.md)
- [Phase 3 Details](./phase-3-<name>.md)

## Phase Summary

| Phase | Name | Tasks | Done | Progress |
|:------|:-----|------:|-----:|:---------|
| 0     | Requirement Clarification | N | 0 | ⬜⬜⬜⬜⬜ |
| 1     | Compliance & Design       | N | 0 | ⬜⬜⬜⬜⬜ |
| 2     | Implementation & Testing  | N | 0 | ⬜⬜⬜⬜⬜ |
| 3     | Delivery                  | N | 0 | ⬜⬜⬜⬜⬜ |

## Phase Checklist
- [ ] Phase 0: <name> (0/N tasks) -- [details](./phase-0-<name>.md)
- [ ] Phase 1: <name> (0/N tasks) -- [details](./phase-1-<name>.md)
- [ ] Phase 2: <name> (0/N tasks) -- [details](./phase-2-<name>.md)
- [ ] Phase 3: <name> (0/N tasks) -- [details](./phase-3-<name>.md)

## Current Status
**Active Phase**: Phase N
**Active Task**: Task description
**Blockers**: None / description

## Next Steps
1. ...
2. ...

## Session Log
| Date | Session | Summary |
|:-----|:--------|:--------|
|      |         |         |
```

## Per-Phase Detail Template (`phase-N-<name>.md`)

```markdown
# Phase N: <Phase Name>

**Goal**: What this phase achieves
**Status**: Not Started / In Progress / Complete

## Tasks
- [ ] **Task N.1**: Description
  - Priority: P0
  - Effort: M
  - Acceptance: How to verify this is done
  - Notes: _none yet_
- [ ] **Task N.2**: Description
  - Priority: P1
  - Effort: S
  - Acceptance: How to verify this is done
  - Notes: _none yet_

## Phase Notes
<!-- Decisions, blockers, and context discovered during this phase -->

## Phase Completion Checklist
- [ ] All tasks above are checked off
- [ ] MASTER.md phase count updated
- [ ] MASTER.md current status moved to next phase
```
